NC News API Documentation

Welcome to the official NC News REST API reference.

Introduction

The NC News API is organized around REST. Our API has predictable resource-oriented URLs, accepts request bodies, returns JSON responses, and uses standard HTTP response codes, and verbs.

Errors

NC News uses conventional HTTP response codes to indicate the success or failure of an API request. In general: Codes in the 2xx range indicate success. Codes in the 4xx range indicate an error that failed given the information provided (e.g., a required parameter was omitted or invalid request body etc.). Codes in the 5xx range indicate an error with NC News’s servers (these are rare).

Retrieve Topics

GET /api/topics

Returns a list of all topics.

Request


curl -X GET "https://nc-news-backend-02ex.onrender.com/api/topics" \
  -H "Accept: application/json"

Response (200 OK)

{
    "topics": [
        {
            "slug": "coding",
            "description": "Code is love, code is life"
        },
        {
            "slug": "football",
            "description": "FOOTIE!"
        }
    ]
}

Retrieve Articles

GET /api/articles

Retrieve a list of articles with optional sorting, filtering, and ordering.

Query Parameters

  • ?topic=coding
  • ?sort_by=created_at (default)
  • ?order=asc or desc (default)

Request


curl -X GET "https://nc-news-backend-02ex.onrender.com/api/articles?sort_by=votes&order=asc" \
  -H "Accept: application/json"

Retrieve Article By Id

GET /api/articles/:article_id

Returns an object of article.

Request


curl -X GET "https://nc-news-backend-02ex.onrender.com/api/articles/1" \
  -H "Accept: application/json"
